# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. Do **not** open a public issue
2. Email the maintainer with details of the vulnerability
3. Include steps to reproduce if possible

## Security Practices

- No secrets or credentials are committed to this repository
- Dependencies are scanned weekly via Dependabot and Trivy
- Static analysis runs on every PR via Semgrep
- Environment variables are used for all sensitive configuration
